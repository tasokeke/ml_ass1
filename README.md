# NaN მნიშვნელობების დამუშავება
Numerical სვეტები - median
Categorical სვეტები - mode (ანუ most_frequent)

Numerical სვეტებთან ვიყენებთ median-ს, რადგან მონაცემები ხშირად შეიცავს outliers-ს, რომლებიც განსაზღვრავენ mean-ს, საშუალო მაჩვენებელს, მაშინ, როცა median უფრო განაზოგადებს და ტიპური სახლის მაგალითს წარმოგვიჩენს. შესაბამისად, ეს უფრო საიმედოა nan-ების შესავსებად.

Categorical სვეტებისთვის გამოვიყენებთ - mode-ს (ანუ most_frequent). კატეგორიული ცვლადი საშუალოდ ვერ შეფასდება (მაგ. ნახევრად ქვა/ნახევრად აგური), ამიტომ ყველაზე ხშირად არსებულ feature-ს მივანიჭებთ missing value-ს, mode-ით შევავსებთ.

# ახალი feature-ების შექმნა

ძველ feature-ებზე დაყრდნობით შევქმენი ისეთი ახალი feature-ები, რომლებიც უფრო მეტად წარმოჩანეს სახლის ფასეულობას.

TotalSF/TotalBathrooms/TotalBsmt- მყიდველები იმით უფრო ინტერესდებიან ჯამურად რამდენია ფართი და არა იმით, რამდენია მაგ. სხვადასხვა სართულის ფასი.
ასეთი მონაცემები high-signal size მეტრიკს ქმნიდა.

Age features - HouseAge/RemodAge/GarageAge
ის უფრო გვაინტერესებს “რამდენი წლისაა” სახლი ან რამდენი წელია მისი რემონტიდან გასული, ვიდრე ის კონკრეტული წელი, როცა აშენდა/გარემონტდა. ეს უფრო მოსახერხებელია შეფასებისას. 

Qual_x_GrLivArea – Synergy-ს გამოხატავს. დიდი სახლი კიდევ უფრო ღირებულია, თუ მისი ხარისხი ძალიან კარგია და არა მაგალითად, მხოლოდ დამაკმაყოფილებელი. ეს ახალი ცვლადი ხელს უწყობს მოდელს, რომ ფართის რეალური ღირებულება უკეთესად განსაზღვროს.

RoomsPerArea - ოთახის სიმჭიდროვე, რომელსაც ფართის ფანლაგების ეფექტურობის შესაფასებლად გამოვიყენებთ

# skewed სვეტები

ის numerical სვეტები, რომელთა skewed არის 0.75-ზე მეტი - np.log1p-ით გავალოგარითმე, რითაც ნორმალურ განაწილებასთან დაახლოვდა extreme value-ებიც. ამით რეალურად ხელი შევუწვეთ linear model-ებს, მათთვის უფრო მეტად ხელმისააწვდომი მონაცემთა განაწილება ჩამოვაყალიბეთ.


# სტანდარტიზაცია

ყველა numerical სვეტი დავასკალირე ისე, რომ საშუალო გამხდარიყო ნული და სტანდარტული გადახრა 1. ეს ridge და lasso მოდელებს გამოადგებათ, ახლა უკვე თანაბარი „ძალა“ ექნება ყველა სვეტს. 


# feature Selection 

feature selection დავყავით ორ ძირითად stage-დ.

პირველია - pearson correlation filter. 
თითოეულ feature-სა და დალოგარითმებულ saleprice-ს შორის გავაკეთე კორელაციის ფილტრი. ყველა ის feature, რომელთა კორელაცია < 0.01 წავშალე. ფაქტობრივად ეს ის სვეტებია, რომლებსაც ფასთან თითქმის არ აქვთ წრფივი დამოკიდებულება.
მეორე stage-ია - rfe. ეს როგორც რაღაც ჯგუფს, ისე უყურებს, feature-ებს და თითოეულ იტერაციაში ყველაზე ნაკლებად საჭიროს შლის, სანამ 80 ყველაზე მნიშვნელოვან სვეტს არ დატოვებს.

ფილტრაცია უფრო სწრაფია, rfe შედარებით ნელი, მაგრამ მეტი სიზუსტით გამოირჩევა, შედეგად თან სწრაფად და თან ეფექტურად ვიღებთ საჭირო data-ს.


# training

# ტესტირებული მოდელები
გვაქვს ექსპერიმენტები სამი სხვადასხვა ჯგუფის მოდელებისთვის.
	
linear models - 
გვაქვს სამი სხვადასხვა ტიპის linear regression.
ჩვეულებრივი (baseline) linear regression, ყოველგვარი რეგულარიზაციის გარეშე და ორი ისეთი ტიპის მოდელი, რომლებიც იყენებენ “ჯარიმებს” - Ridge regression (l2 penalty) - 0.1, 1.0, 10.0, 50.0 და 100.0 ალფებზე და Lasso regression (L1 penalty) 0.0001, 0.001, 0.01, 0.1 და 1.0 ალფებზე.

decision trees -
გამოვიყენეთ სამ სხვადასხვა ჰიპერპარამეტრზე max_depth, min_samples_split, max_features დატრენინგებული decision tree-ები იმისათვის, რომ დაგვენახა რა გავლენა აქვს ხის სირთულეს under/overfitting-ზე.

random forests -
random forest-ებიც სამ სხვადასხვა პარამეტრზე n_estimators, max_depth, min_samples_leaf დაიტესტა. 


# hyperparameter ოპტიმიზაციის მიდგომა

hyperparameter-ების ოპტიმიზაციისთვის გამოვიყენეთ One-Axis-at-a-Time მეთოდი, ანუ უფრო მარტივად რომ ვტქვათ, თითოეული  hyperparameter ვცვალეთ წინასწარ განსაზღვრულ დიაპაზონში, დანარჩენები მნიშვენლობები კი ფიქსირებული იყო გონივრულ, დადგენილ ფარგლებში. ეს მიდგომა შეგნებულად იქნა არჩეული ნაცვლად ავტომატიზებული (მაგ. GridSearchCV) მეთოდებისა და ამას რამდენიმე მიზეზი აქვს.

მთავარი მიზეზი კი, რა თქმა უნდა ისაა, რომ ექსპერიმენტის მთავარი მიზანი არ იყო მხოლოდ საუკეთესო მოდელის პოვნა, არამედ იმის გაგება, თუ როგორ მოქმედებს თითოეული hyperparameter მოდელის ქცევაზე, underfitting-სა და overfitting-ზე დაკვირვება. Grid search ერთდროულად რამდენიმე პარამეტრს ცვლის, ამიტომ ამ ეფექტებს ნათლად ვერ დავინახავდით.
ამავდროულად, შეფასებისთვის გამოყენებული validation set იყო ფიქსირებული, ისაა შექმნილი ნებისმიერი ტრენინგის დაწყებამდე. ამის წყალობით,  არცერთი hyperparameter-ის გადაწყვეტილება არ იყო ინფორმირებული test set-ის მიერ და ყველა run შედარებადი იყო ერთი და იმავე მონაცემებზე.

# linear models
	
ჩვეულებრივი (უჯარიმო) linear regression აღწევს - train=0.1127, val=0.1322-ს, 0.0195 გეფით, რაც არც ისე ცუდი დასაწყისია, მხოლოდ მცირე overfitting გვაქვს. ეს იმის მაუწყებელი უნდა იყოს, რომ ჩვენმა feature engineering-მა დადებითად იმოქმედა data-ზე. ridge აჩვენებს პროგრესს თავისი ჯარიმის საშუალებით. 0.1-დან 100-მდე იცვლება ალფა და ვალიდაციის rsme-ც იკლებს 0.1321-დან 0.1299-მდე, ფეგიც სხვაობა 0.0194-დან 0.0161-მდე მცირდება. ანუ პროგრესი არის,  მაგრამ ეს მოდელები ყველა კოეფიციენტს თანაბრად ამცირებენ ყოველფგვარი აღმოფხვრის გარეშე, ამიტომ მას უნდა გაუჭირდეს feature-ების შერჩევა მაღალკორელაციური მახასიათებლებისთვის. არც ეს გვაწყობს. lasso ძალიან განსხვავდება სხვადასხვა ალფისთვის. ალფა=0.0001-ზე მოდელი არსებითად არარეგულარიზებულია (val=0.1315). ალფა=0.001-ზე L1 ჯარიმა საუკეთესო წერტილს აღწევს: ვალიდაციის rmse ეცემა 0.1278-მდე მხოლოდ 0.0142 გეფით. მაგრამ შემდეგ ალფა=0.01-ზე ტრენინგის შეცდომა იწყებს ზრდას და ვალიდაციის გეფიც იზრდება ალფა=0.1-ისა და ალფა=1.0-ის მიხედვით, მოდელი აშკარა underfitting განიცდის. Gap აღწევს 0.207-სა და 0.390-ს, რაც ვალიდაციის error-ს 0.225-მდე და 0.433-მდე ამცირებს. L1 ჯარიმა ძალიან ბევრი კოეფიციენტისთვის ნულია, რაც მოდელს ართმევს იმის საშუალებას, რომ მონაცემებს მთლიანად მოერგოს. ანუ ესაა underfitting-ის მაგალითი ზედმეტი რეგულარიზაციის გამო.

# decision tree
	
depth sweep ყველაზე კარგად გვიჩვენებს გზას underfitting-დან overfitting-მდე
depth=3-ის დროს მოდელი აღწევს ისეთ მაჩვენებლებს, როგორიცაა train=0.1974, val=0.2260. ორივე მათგანზე  აქამდე არსებულ საუკეთესო სედეგთან შედარებით მაღალი ცდომილება მიუთითებს underfitting-ზე. ხე ზედმეტად მარტივია (shallow), რათა აღიქვას მონაცემებში არსებული არაწრფივი კავშირები.სიღრმის 5-მდე და 8-მდე გაზრდით, ვალიდაციის ცდომილება საგრძნობლად უმჯობესდება (0.189 და 0.184), ხოლო სხვაობა წვრთნასა და ვალიდაციას შორის (0.042 და 0.097) ჯერ კიდევ კონტროლირებადია. თუმცა, depth=12-დან და ზემოთ, სიტუაცია მკვეთრად იცვლება. Train-ის ცდომილება „კოლაფსირებს“ 0.027-მდე (depth=12) და 0.002-მდე (depth=20), მაშინ როდესაც ვალიდაციის ცდომილება რეალურად იზრდება 0.202-მდე და 0.204-მდე. სხვაობა (gap) ადის 0.175 და 0.202-მდე — ხემ სრულად დაიზეპირა საწვრთნელი ნაკრები და დაკარგა განზოგადბის უნარი. 

min_samples_split sweep განსხვავებულად მუშაობს. ამ მონაცემის 2-დან 10-მდე და 50-მდე გაზრდა ამცირებს overfitting-ს (gap მცირდება და val rsme ნარჩუნდება), მაგრამ მოდელი სანდო მაინც ვერაა, რადგან 100-ზე ისევ იწყებს underfitting-ს.

max_features ექსპერიმენტები ორივე rsme-ს საკმაოდ მაღალს აჩვენებენ სხვა მოდელებთან შედარებით. ამის მიზეზი ისაა, რომ რენდომ ხდება გაყოფა და როგორც წესი არც თუ ისე საუკეთესო მონაცემებს აქცევს მოდელი ყურადღებას და train-ც და validation-ც ორივე იჩაგრება.

თუ მხოლოდ ეს მოდელი გვექნებოდა საუკეთესო ბალანსს მივაღწევდით depth=8-ისა და საშუალო min_samples_split-ის (როცა არც underfitting გვქონდა და არც overfitting) პირობებში.

# random forest

აქ ძირითადად ვხედავთ high variance-ს. rf სტაბილურად აღწევს ძალიან დაბალ training error-ს (∼0.052), მაგრამ ახასიათებს მუდმივი train-val gap (∼0.09–0.10). ეს ასახავს იმ ზომიერ overfitting-ს, რომელიც თანდაყოლილია ღრმა, unpruned ხეებისთვის — მაშინაც კი, როდესაც ისინი ensemble-ში არიან გაერთიანებულები.
ხეების რაოდენობის უბრალო მატებას თითქმის არანაირი სარგებელი არ მოაქვს generalization-ისთვის: val error ძლივს იცვლება. ensemble სტაბილურდება 100 ხემდეც კი და შემდგომი გამოთვლები არაფერს ახალს არ გვმატებს.

max_depth: ეს sweep ადასტურებს, რომ rf უკვე თავისი variance ceiling-ის მახლობლადაა სტანდარტული settings-ებით: 10-დან None-მდე ყველა depth-ი იძლევა validation error-ს 0.1483-სა და 0.1490-ს შორის.

min_samples_leaf: ეს ექსპერიმენტები ყველაზე მეტ ინფორმაციას იძლევა. leaf size-ის გაზრდა 1-დან 16-მდე სტაბილურად ამცირებს train-val gap-ს (0.096-დან 0.033-მდე), თუმცა ეს ხდება validation error-ის პირდაპირი ზრდის ხარჯზე (0.1486-დან 0.1682-მდე).

ტყე ახდენს variance-ის კონვერტაციას bias-ში, მაგრამ ეს bias-variance tradeoff მაინც არაა საუკეთესო, მართლაც rf-ის შედეგი საერთო ჯამში (rf_nestimators_400, val=0.1472) მაინც საგრძნობლად ჩამორჩება საუკეთესო წრფივ მოდელს.


decision tree მოდელები over/underfitting გარდა იმიტომაც უარვყავით, რომ უფრო მაღალი val rsme ჰქონდათ, ვიდრე დანარჩენებს. ხოლო rf-ს linear models იმიტომ ვარჩიეთ, რომ მისი ensemble საშუალო მაჩვენებელი სრულად ვერ დააკომპენსირერბდა ინდივიდუალური ხეების მიდრეკილებას overfitting-ისკენ. data set-ში არის ძალიან ძლიერი წრფივი დამოკიდებულება ჩვენს შესაფასებელ ფასსა და ისეთ features-ს შორის, როგორიცაა GrLivArea, OverallQual და TotalBsmtSF. ამიტომაც ვარჩიეთ მოდელი, რომელიც წრფივი იყო და თან შესაბამისი რეგულარიზაციის წყალობით, ყველაზე დაბალი val rsme-ც მოგვცა.
